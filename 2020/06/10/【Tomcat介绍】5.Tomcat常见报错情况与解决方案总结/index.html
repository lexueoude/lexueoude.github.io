如果按照之前我们笔记的配置仍不能成功的话，一般来说有以下几种情况：

1、java配置路径有问题，具体参考我们java入门阶段的路径与环境变量的配置

2、项目未添加tomcat驱动， （一般提示The superclass “javax.servlet.http.HttpServlet” was not found on the Java Build Path属于这一类）

参考解决方法：保证已经装了tomcat的情况下，eclipse中选择对应项目右键build path ->Configure build path -> Java build path->选择Libraries ->点击右方Add Library… ->Server Runtime ->选择自己的tomcat服务器->Finish

3、项目中的web.xml中配置的servlet的名称写错，tomcat无法识别， （一般控制台提示java.util.concurrent.ExecutionException属于这种问题）

参考解决方法：检查自己配置的servlet名称和路径是否正确。

4、端口被占用，一般由于上一次非正常关闭tomcat或eclipse导致，（ 一般控制台会报：Port 8080 is already in use）

参考解决方法：

1.修改端口号（一般情况下我们不建议修改端口,如果一定要去修改，可以在Tomcat根目录（/conf/Server.xml 里面修改）

2.关闭被占用的端口进程:如果是windows系统，打开CMD输入

1）netstat -ano|findstr 8080看到所有使用的进程，

2）taskkill /pid 8080 /f （这里填写进程信息中最后一个数字编号）

3）重启Tomcat
